    📄 HTML Fundamentals HTML

    ▼

    HTML (HyperText Markup Language) is the skeleton of every webpage. It uses a system of tags to give structure and meaning to content, telling browsers what everything is — not how it looks.

    Document Structure

    Required

    Every HTML document begins with a <!DOCTYPE html> declaration telling the browser to use the modern HTML5 standard. The <html> root element wraps everything. Inside, <head> holds invisible metadata and <body> holds visible content.

    HTML 
    <!-- Every HTML file starts with this -->
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document Title</title>
</head>
<body>
  <!-- All visible content goes here -->
</body>
</html>

    Links & Navigation

    Navigation

    The anchor <a> element creates hyperlinks. The href attribute sets the destination — an absolute URL, relative path, or #id fragment. Add target="_blank" to open in a new tab; always pair it with rel="noopener noreferrer" for security.

    HTML
    <!-- External link -->
<a href="https://example.com" target="_blank" rel="noopener noreferrer">
  Visit Example
</a>

<!-- Same-page anchor -->
<a href="#section-id">Jump to Section</a>

<!-- Email / phone -->
<a href="mailto:hello@example.com">Email Us</a>
<a href="tel:+1234567890">Call Us</a>

    🎨 CSS Fundamentals CSS

    ▼

    CSS (Cascading Style Sheets) controls the visual presentation of HTML. Three fundamental concepts govern how styles are applied: the Cascade, Specificity, and Inheritance.

    Cascade, Specificity & Inheritance

    Core

    When multiple rules target the same element, specificity determines which wins. Think of it as a score: inline style (1000) > #id (100) > .class / [attr] / :pseudo (10) > element (1).

    The Cascade then resolves ties by origin (user-agent → user → author) and source order — later rules win. Inheritance means children automatically receive certain parent properties (color, font-family) but not others (margin, border).

    Specificity Score Visual
    0
    *
    1
    element
    10
    .class
    100
    #id
    1000
    inline
    (0, 0, 0)
    CSS
    /* Specificity examples */
*           { color: black; }     /* 0,0,0 */
p           { color: gray; }      /* 0,0,1 */
.text       { color: blue; }      /* 0,1,0 */
p.text      { color: green; }     /* 0,1,1 */
#header     { color: purple; }    /* 1,0,0 */

/* Inline style beats all (unless !important) */
/* <p style="color:red"> → wins */

/* !important breaks the cascade (use sparingly) */
p { color: red !important; }
    Try It — CSS Syntax

    Three Ways to Add CSS

    Basics

    External stylesheets are best for production — they cache separately and keep concerns separated. Internal <style> blocks suit small prototypes. Inline styles have highest specificity but are impossible to maintain at scale.

    HTML + CSS
    <!-- 1. External (recommended) -->
<link rel="stylesheet" href="style.css">

<!-- 2. Internal -->
<style>
  body { margin: 0; }
</style>

<!-- 3. Inline (avoid for layout) -->
<p style="color: red; font-size: 18px;">Text</p>

    🔧 CSS Variables (Custom Properties)

    ▼

    CSS Custom Properties (variables) store reusable values. Define them with --name: value, access them with var(--name). They cascade, inherit, and can be changed with JavaScript — enabling powerful theming.

    Defining & Using Variables

    Modern

    Declare global variables on :root (the <html> element). Override them on any descendant — this is how dark/light mode toggling works on this very page!

    CSS
    /* Define globally */
:root {
  --color-primary: #7B2FBE;
  --color-accent:  #D4AF37;
  --font-body:     'EB Garamond', serif;
  --spacing-md:    16px;
  --radius:        8px;
}

/* Use them */
.button {
  background:    var(--color-primary);
  color:         var(--color-accent);
  padding:       var(--spacing-md);
  border-radius: var(--radius);
}

/* Dark mode override */
[data-theme="dark"] {
  --bg: #0E0818;
  --text: #FAF5E4;
}

/* Fallback value if var not defined */
p { color: var(--text-color, black); }

    📦 Box Model

    ▼

    Every HTML element is a rectangular box. The box model defines its four layers: content, padding, border, and margin. Understanding this is fundamental to all CSS layout.

    The Four Layers

    Layout
    margin
    border
    padding
    content
    CSS
    /* The box-sizing fix — use everywhere */
* { box-sizing: border-box; }
/* border-box includes padding+border in width */

.box {
  width:   300px;      /* content width */
  height:  200px;

  padding: 20px;       /* all sides */
  padding: 10px 20px;  /* top/bottom left/right */
  padding: 5px 10px 15px 20px; /* T R B L (clockwise) */

  border:       2px solid gold;
  border-radius: 8px;

  margin:   20px;        /* outer spacing */
  margin:   0 auto;      /* center horizontally */
}

    🔲 CSS Grid

    ▼

    CSS Grid is a two-dimensional layout system — it handles both rows AND columns simultaneously. Best for page-level layouts, dashboards, and complex card arrangements.

    Grid Layout

    Layout
    Live Grid Demo
    Header (span 3)
    Card 1

    Card 3 (span 2)
    Card 4

    CSS
    /* Container */
.grid {
  display:               grid;
  grid-template-columns: repeat(3, 1fr);    /* 3 equal columns */

  grid-column: 1 / -1;   /* span full width */

    ✨ CSS Animations

    ▼

    CSS animations let you animate any CSS property over time using @keyframes rules. Unlike transitions (triggered by state changes), animations run automatically and can loop.

    @keyframes & Animation Properties

    Motion
    Live Animation Demo
    spin
    pulse
    slide
    bounce
    CSS
    /* 1. Define keyframes */
@keyframes slideIn {
  from { transform: translateX(-100%); opacity: 0; }
  to   { transform: translateX(0);     opacity: 1; }
}

@keyframes pulse {
  0%, 100% { transform: scale(1); }
  50%       { transform: scale(1.2); }
}

/* 2. Apply to element */
.box {
  animation-name:            slideIn;
  animation-duration:        0.5s;
  animation-timing-function: ease-out;
  animation-delay:           0.1s;
  animation-iteration-count: 1;      /* or infinite */
  animation-direction:       normal;  /* alternate, reverse */
  animation-fill-mode:       both;    /* forwards, backwards */

  /* Shorthand: name duration timing delay count direction fill */
  animation: slideIn 0.5s ease-out 0.1s 1 normal both;
}

/* Prefer reduced motion for accessibility */
@media (prefers-reduced-motion: reduce) {
  * { animation-duration: 0.01ms !important; }
}

    CSS Math Functions

    Functions
    CSS
    /* calc() — mix units */
.sidebar { width: calc(100% - 240px); }

    🏆 Best Practices

    ▼

    Great CSS isn't just working CSS — it's organized, performant, and maintainable CSS that your future self will thank you for.

    BEM Naming Convention

    Architecture

    BEM (Block, Element, Modifier) gives a systematic naming convention. It prevents style conflicts and makes CSS self-documenting.

    CSS + HTML
    /* Block = standalone component */
.card { }

/* Element = part of Block (double underscore) */
.card__title  { }
.card__image  { }
.card__footer { }

/* Modifier = variation of Block or Element (double dash) */
.card--featured       { }
.card--dark           { }
.card__title--large   { }

<!-- In HTML -->
<div class="card card--featured">
  <img class="card__image" src="...">
  <h2 class="card__title card__title--large">Title</h2>
</div>

    ↕️ CSS Logical Properties Modern

    ▼

    Logical properties use writing-mode-relative directions instead of physical (top/right/bottom/left), making your CSS work correctly in all languages including RTL (Arabic, Hebrew) and vertical (Japanese) text.

    Physical → Logical Mapping

    i18n
    Physical (old)Logical (new)Meaning
    widthinline-sizeSize along writing direction
    heightblock-sizeSize perpendicular to writing
    margin-topmargin-block-startStart of block axis
    margin-bottommargin-block-endEnd of block axis
    margin-leftmargin-inline-startStart of inline axis
    margin-rightmargin-inline-endEnd of inline axis
    padding-top/bottompadding-blockBoth block sides
    padding-left/rightpadding-inlineBoth inline sides
    border-topborder-block-start—
    top/bottominset-blockFor positioned elements
    left/rightinset-inlineFor positioned elements
    text-align: lefttext-align: startWriting-mode aware
    CSS
    /* Old way */
.card {
  margin-top: 16px;
  margin-bottom: 16px;
  padding-left: 24px;
  padding-right: 24px;
}

/* New logical way (RTL-friendly!) */
.card {
  margin-block:  16px;
  padding-inline: 24px;
  border-inline-start: 3px solid gold; /* left in LTR, right in RTL */
  inset-inline-start: 0;              /* replaces 'left' */
}

    🪆 Native CSS Nesting CSS 2024

    ▼

    CSS now supports native nesting — no preprocessor needed. Nest selectors inside each other just like Sass/SCSS.

    CSS Nesting Syntax

    Modern
    CSS (Native)
    .card {
  background: white;
  padding: 20px;

  /* Nested selectors */
  & .title {              /* .card .title */
    font-size: 1.2rem;
  }

  &:hover {               /* .card:hover */
    box-shadow: 0 4px 20px rgba(0,0,0,0.2);
  }

  &.featured {           /* .card.featured */
    border-color: gold;
  }

  /* Nested media query */
  @media (max-width: 600px) {
    padding: 12px;
  }
}

    🗂️ CSS @layer — Cascade Layers CSS 2024

    ▼

    @layer lets you explicitly control the cascade order of styles — no more specificity wars. Later layers win over earlier ones, regardless of selector specificity.

    @layer Usage

    Modern
    CSS
    /* Declare layer order (lowest to highest priority) */
@layer reset, defaults, components, utilities;

/* Styles in earlier layers can be overridden by later layers */
@layer reset {
  * { margin: 0; box-sizing: border-box; }
}

@layer components {
  .btn { padding: 8px 16px; background: purple; }
}

@layer utilities {
  /* Even a low-specificity class here beats .btn */
  .p-0 { padding: 0 !important; }
}

/* Import a stylesheet into a layer */
@import url('bootstrap.css') layer(vendor);

  justify-content: center;

@each $size in sm, md, lg {
  .text-#{$size} { font-size: map-get($sizes, $size); }
